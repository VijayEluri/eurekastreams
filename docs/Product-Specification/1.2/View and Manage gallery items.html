View and Manage gallery items

Follow and Subscribe to Streams View and Search a Stream

Overview

Gallery can be updated with default apps, themes and tabs. Eureka administrators are responsible for vetting the item available within the gallery for their employees.

Conditions of satisfaction

Browse

  1. The knowledge worker must be able to browse a list of apps, themes and tabs by recently published, popularity or category.
  2. The knowledge worker must be able to view an app’s title, thumbnail, website URL, description, user count, author, category, and publish date.
  3. The knowledge worker must be able to view a theme’s title, thumbnail, website URL, description, user count, author, category, and publish date.
  4. The knowledge worker must be able to view a tab’s title, description, user count, category, and publish date.
  5. The knowledge worker must be able to copy a snapshot of a tab in the gallery onto his start page when adding a tab.

Manage apps

  1. The Eureka administrator must be able to add apps that adhere to the Google App Specification to the gallery
  2. The Eureka administrator must be able to specify category for any app in the gallery.
  3. The Eureka administrator must be able to remove apps from the gallery.
  4. The Eureka administrator must be able to modify the URL for a published app.

Manage themes

  1. The Eureka administrator must be able to add themes that adhere to the theme schema to the gallery
  2. The Eureka administrator must be able to specify category for any theme in the gallery.
  3. The Eureka administrator must be able to remove themes from the gallery.
  4. The Eureka administrator must be able to modify the URL for a published theme.

Manage tabs

  1. The Eureka administrator must be able to add any tab on his start page to the gallery.
  2. The Eureka administrator must be able to specify category for any tab in the gallery.
  3. The Eureka administrator must be able to remove tabs from the gallery.
  4. The Eureka administrator must be able to specify and modify a description for a published tab.

User Experience

Browse

Browsing apps, themes and tabs

  1. Displays available apps
  2. Displays available themes
  3. Displays available tabs
  4. Sorts by the most recently added
  5. Sorts by the most popular
  6. Sorts by category

Viewing the details of an app

  1. Title
  2. Thumbnail
  3. Description
  4. Website URL
  5. Category, User count, and Author
  6. Publish Date

Viewing the details of a theme

  1. Title
  2. Thumbnail
  3. Description
  4. Category, User count, Author, Published Date

Viewing the details of a tab

  1. Title
  2. Description
  3. Category, User count, Publish Date

Manage

Viewing and Managing Apps

  1. Opens the add app form.
  2. Hovering over the app thumbnail, title, description or meta data displays the edit link. Clicking edit displays the edit app form.
  3. Hovering over the app thumbnail, title, description or meta data displays the Delete link. Clicking the Delete link removes the app from the gallery. When the Eureka administrator clicks the Delete link a popup window is displayed asking the user: “Are you sure you want to delete this app? Deleting this app will remove it from the start page of every user that has added it to a tab.” If the app is deleted a feedback message is displayed saying: “The [app name] app has been deleted”.
  4. Adds the app to the user’s start page.

Add an app to the gallery

  1. Drop down menu containing a list of app categories
  2. Input field for app XML address
  3. Submits the form and returns the Eureka administrator back to the configure page defaulted to the all apps view. A feedback message “Your app has been successfully added” is also displayed at the top of the page with a close link next to it.
  4. Discards the changes made to the form and returns the Eureka administrator back to the configure page.

Editing and removing apps

  1. Populated with the app category.
  2. Populated with URL to app XML file.
  3. Saves changes to the form and returns the Eureka administrator back to the configure page defaulted to the all apps view. A feedback message “Your app has been successfully saved” is also displayed at the top of the page with a close link next to it.
  4. Discards the changes made to the form and returns the Eureka administrator back to the configure page.

Managing and Viewing Themes

  1. Opens the add theme form.
  2. Hovering over the theme thumbnail, title, or meta data displays an edit link. Clicking edit displays the edit theme form.
  3. Hovering over the theme thumbnail, title, or meta data displays the Delete link. Clicking the Delete link removes the theme from the gallery. When the Eureka administrator clicks the Delete link a popup window is displayed asking the user: “Are you sure you want to delete this theme? Deleting a theme will remove it from the start page and apply the default theme for every user that has applied it.” If the theme is deleted a feedback message is displayed saying: “The [theme name] theme has been deleted” with a close link next to it.
  4. Applys the theme to the user’s start page.

Add a Theme to the gallery

  1. Drop down menu containing a list of theme categories
  2. Input field for theme XML address
  3. Submits the form and returns the Eureka administrator back to the configure page defaulted to the all themes view. A feedback message “Your theme has been successfully added” is also displayed at the top of the page with a close link next to it.
  4. Discards the changes made to the form and returns the Eureka administrator back to the configure page.

Editing and removing a theme

  1. Populated with the theme category.
  2. Populated with URL to theme XML file.
  3. Saves changes to the form and returns the Eureka administrator back to the configure page defaulted to the all themes view. A feedback message “Your theme has been successfully saved” is also displayed at the top of the page with a close link next to it.
  4. Discards the changes made to the form and returns the Eureka administrator back to the configure page.

Managing and viewing a tab

  1. Opens the add theme form.
  2. Hovering over the theme thumbnail, title, or meta data displays an edit link. Clicking edit displays the edit theme form.
  3. Hovering over the theme thumbnail, title, or meta data displays the Delete link. Clicking the Delete link removes the theme from the gallery. When the Eureka administrator clicks the Delete link a popup window is displayed asking the user: “Are you sure you want to delete this theme? Deleting a theme will remove it from the start page and apply the default theme for every user that has applied it.” If the theme is deleted a feedback message is displayed saying: “The [theme name] theme has been deleted” with a close link next to it.
  4. Adds the tab to the user’s start page.

Add a tab to the gallery

  1. Drop down menu containing a list of tab’s from the Eureka administrator’s start page.
  2. Description of the tab.
  3. Drop down menu containing a list of theme categories
  4. Submits the form and returns the Eureka administrator back to the configure page defaulted to the all tabs view. A feedback message “Your tab has been successfully added” is also displayed at the top of the page with a close link next to it.
  5. Discards the changes made to the form and returns the Eureka administrator back to the configure page.

Editing and removing a tab

  1. Drop down menu containing a list of tab’s from the Eureka administrator’s start page.
  2. Description of the tab.
  3. Drop down menu containing a list of theme categories
  4. Submits the form and returns the Eureka administrator back to the configure page defaulted to the all tabs view. A feedback message “Your tab has been successfully saved” is also displayed at the top of the page with a close link next to it.
  5. Discards the changes made to the form and returns the Eureka administrator back to the configure page.

Test Plan

Browse

  1. Verify a knowledge worker can browse a list of apps by recently published, popularity or category
    • Verify the ability to browse recently published apps in the gallery
    • Verfiy the publish dated reflect this view
    • Verify the ability to add an app from the recent view
    • Verify the ability to browse apps in the gallery by popularity
    • Verify the user counts reflect this view
    • Verify the ability to add an app from the popular view
    • Verify the ability to browse apps in the gallery by category
    • Verify the ability to add an app from a specific category
  2. Verify a knowledge worker can browse a list of themes by recently published, popularity or category
    • Verify the ability to browse recently published themes in the gallery
    • Verfiy the publish dated reflect this view
    • Verify the ability to add a theme from the recent view
    • Verify the ability to browse themes in the gallery by popularity
    • Verify the user counts reflect this view
    • Verify the ability to add a theme from the popular view
    • Verify the ability to browse themes in the gallery by category
    • Verify the ability to add a theme from a specific category
  3. Verify a knowledge worker can browse a list of tab by recently published, popularity or category
    • Verify the ability to browse recently published tabs in the gallery
    • Verfiy the publish dated reflect this view
    • Verify the ability to add a tab from the recent view
    • Verify the ability to browse tabs in the gallery by popularity
    • Verify the user counts reflect this view
    • Verify the ability to add a tab from the popular view
    • Verify the ability to browse tabs in the gallery by category
    • Verify the ability to add a tab from a specific category
  4. Verify a knowledge worker can view an app’s attributes
    • Verify the title, thumbnail, website URL, description, user count, author, category and date published are present
    • Verify all of these attributes are present for all apps in the gallery
  5. Verify a knowledge worker can view a theme’s attributes
    • Verify the title, thumbnail, website URL, description, user count, author, category and date published are present
    • Verify all of these attributes are present for all themes in the gallery
  6. Verify a knowledge worker can view a tab’s attributes
    • Verify the title, description, user count, category and date published are present
    • Verify all of these attributes are present for all tab’s in the gallery
  7. Verify a knowledge worker can confirm the ability to copy a snapshot of a tab from the gallery to a start page tab
    • Verify all of the correct apps are preset on the added tab

Manage Apps

  1. Verify a Eureka administrator can add apps to the gallery
    • Verify there is a link from the gallery directing the user to a form to upload an app
    • Verify error handling is present when required fields are not populated
    • Verify a category can be selected and associated with the app
    • Verify a theme that is uploaded must adhere to the Google app specification
    • Verify that if a app does not adhere there is error handling
    • Verify there is a link which will direct the user to the app specification guide
  2. Verify a Eureka administrator can modify the URL for a published app
    • Verify the category can be changed for an app that has been uploaded to the gallery
    • Verify the URL can be changed for an app that has been uploaded to the gallery
    • Verify there is error handling in the edit mode
  3. Verify a Eureka administrator can remove apps from the gallery
    • Verify a published app can be removed
    • Verify there is removal confirmation
    • Verify the app is removed from the gallery, but is not removed from any tabs of users with that app currently applied
