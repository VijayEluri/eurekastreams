This page will replace the 0.9 specification for [Share, save, flag, or comment on an activity](../0.9/Share,%20save,%20flag,%20or%20comment%20on%20an%20activity.html) when Eureka Stream 1.0 is released.  This product specification is a work in process.  In the 1.0 release, the ability to Like an activity is being added.
## Overview
Knowledge workers can share activity between employee and group streams, save favorite activity, flag activity as inappropriate, like an activity, or add comments to activities.
## Conditions of satisfaction
1. The knowledge worker must be able to share an activity to a colleague, public group, or private group he is a member of.
2. The knowledge worker must not be able to share activity posted to a private group.
3. The knowledge worker must be able to attach a comment to an activity he is sharing.
4. The knowledge worker must be able to save and un-save an activity.
5. The knowledge worker must be able to comment on an activity when he is not restricted by stream moderation.
6. The knowledge worker must be able to delete a comment he has posted.
7. The comment must be restricted to 250 characters with visual feedback when the limit has been reached.
8. The knowledge worker must be able to flag an activity as inappropriate.
9. The knowledge worker must be able to like and unlike an activity.
10. The knowledge worker must be able to view the number and browse the list of knowledge workers who like an activity.
11. The group coordinator must be able to delete any comment posted to his group stream.
12. The organization coordinator must be able to delete any comment posted to a stream associated to his organization or sub-organization.


## User experience

### Comment on an activity

### Share an activity

![](images/R1_PS_F13_sharesaveflag_ShareActivity_fig01.png)

1. A knowledge worker can share an activity by hovering over an activity and clicking on the share link.  Share option is not available for activities posted in a private group.

![](images/R1_PS_F13_actionsonanactivity_ShareActivity_fig02.png)

1. A knowledge worker can share an activity to their own stream or to another individual or group stream.  If posting is turned off, the individual or group should not be displayed in the autocomplete.
2. Activity being shared.
3. Input box to enter a comment.
4. Remaining characters
5. The share button is only active when the remaining character count is less than 250 or 0 or greater.

![](images/PS_F13_sharesaveflag_ShareActivity_fig03.png)

1. Name of the knowledge worker sharing the activity and the stream it is being shared to.
2. Name of the knowledge worker that authored the activity.
3. Comment from the knowledge worker sharing the activity.

### Like an activity

![](images/R1_PS_F13_sharesaveflag_LikeActivity_fig01.png)

![](images/R1_PS_F13_sharesaveflag_LikeActivity_fig02.png)

![](images/R1_PS_F13_sharesaveflag_LikeActivity_fig03.png)

1.  An activity displays it's like count (if over 0) to the knowledge worker. When the knowledge worker clicks on the thumbs up icon or the like button, the user has 'liked' the activity and the number increments.
2. A knowledge worker can hover over the like count showing a hovering card displaying the last 10 people to have liked the activity with an option to view more.
3. A knowledge worker is able to unlike the activity by clicking on 'unlike'.

![](images/R1_PS_F13_actionsonanactivity_like_fig04.png)

1. A knowledge worker will be presented with this modal listing all employees who liked an activity if he/she clicks on 'view all'

### Flag an activity

![](images/R1_PS_F13_sharesaveflag_FlagActivity_fig01.png)

1. A knowledge worker can flag an activity by hovering over an activity and clicking on the flag link. When the knowledge worker clicks the flag link, a pop-up window will be displayed asking them: "Flagged activities will be sent to the organization coordinator for review.  Are you sure you want to flag this activity as inappropriate?" If the knowledge worker clicks yes the activity remains on the page.  Feedback is displayed at the top of the page with a close option: "Activity has been flagged".  Refer to the [Action Feedback UI design pattern](/docs/product-specification/user-experience-guidelines/action-feedback/) for additional details.   The actions described in the [View and manage an organization profile](/docs/product-specification/features/view-manage-org-profile/) Flagged activities section and [Manage and Receive Notifications](/docs/product-specification/features/manage-receive-notifications/) UX designs are performed.

### Save an activity

![](images/R1_PS_F13_sharesaveflag_SaveActivity_fig01.png)

1. A knowledge worker can save an activity by hovering over an activity and clicking on save link.

![](images/R1_PS_F13_sharesaveflag_SaveActivity_fig02.png)

1. When an activity is saved the save link changes to "Unsave" allowing the knowledge worker to remove the activity from their saved items.

## Test plans
### Knowledge worker
1. **Action: Share activities**
    * Verify an activity can be shared in the following cases:
        * colleague to colleague
        * colleague to public group
        * colleague to private group where knowledge worker is a member
        * public group to public group
        * public group to private group where knowledge worker is a member
        * public group to colleague
2. **Action: Attempt to share activity within a private group**
    * Verify share link is not provided for an activity with comments
    * Verify share link is not provided for an activity without comments
3. **Action: Attach comment when sharing an activity**
    * Verify comments can be added when sharing activities
    * Verify character count is limited to 250 characters
    * Verify visual notification indicating character count
4. **Action: Save and un-save an activity**
    * Verify activities can be saved
    * Verify activities can be un-saved
    * Verify that saved activity appears in the Saved items stream
5. **Action: Comment on activity (assuming commenting is enabled)**
    * Verify a comment can be posted against any activity
6. **Action: Delete a comment**
    * Verify a comment can be deleted only by the knowledge worker it was posted by
    * Verify a delete confirmation is presented
    * Verify the delete confirmation must be acknowledged before the delete is processed
7. **Action: Restrict character limits for commenting on activity**
    * Verify commenting is limited to 250 characters
    * Verify visual notification indicating character count
8. **Action: Flag an activity as inappropriate**
    * Verify the ability to flag an activity as inappropriate
9. **Action: Confirm an activity can be liked and unliked**
    * Verify that an activity displays it's like count (if over 0) to the knowledge worker. 
    * Verify that when the knowledge worker clicks on the thumbs up icon or the like button, the user has 'liked' the activity and the number increments.
    * Verify that when a knowledge worker hovers over the like count, a hovering card displays the last 10 people that liked the activity with an option to view more.
    * Verify that when view more is clicked on the hovering card, the full list of colleagues who liked the activity displays.
    * Verify that the knowledge worker is able to unlike the activity by clicking on 'unlike'.
### Group Coordinator
1. **Action: Execute same tests as Knowledge Worker**
    * Verify tests executed successfully
2. **Action: Delete Comments**
    * Verify a Group Coordinator is able to delete any comments in the activity stream of his group
### Organization Coordinator
1. **Action: Execute same tests as Knowledge Worker**
    * Verify tests executed successfully
1. **Action: Delete Comments**
    * Verify an Organization Coordinator is able to delete any comments in the activity stream of his organization or sub-organization
### Root Org Coordinator
1. **Action: Execute same tests as Knowledge Worker**
    * Verify tests executed successfully
2. **Action: Execute same tests as Organization Coordinator**
    * Verify tests executed successfully
