Customize the start page

Configure a stream Login to the system

Overview

The start page serves as a private homepage for the knowledge worker allowing them to organize and find the information they need on a daily basis. The user has access to a gallery where they can choose from a collection of apps that they place on their start page. A knowledge worker can also organize the apps on their start page by grouping them under tabs. Unlike traditional enterprise web portals, knowledge workers not only have complete control over the content but they may also apply a theme to change the look and feel.

Conditions of satisfaction

Themes

  1. The knowledge worker must be able to apply a theme to the start page.

Tabs

  1. The knowledge worker must be able to create up to six tabs on the start page.
  2. The knowledge worker must see a three-column layout as the default when a new tab is created
  3. The knowledge worker must be able to apply a layout to a tab.
  4. The knowledge worker must be able to re-arrange the order of the tabs on the start page.
  5. The knowledge worker must be able to rename a tab.
  6. The knowledge worker must be able to remove a tab unless it is the last one on the start page.
  7. The knowledge worker must be able to undo the removal of a tab.

Apps

  1. The knowledge worker must be able see empty zones on a tab and an explicit action to add an app.
  2. The knowledge worker must be able to add an app to a tab.
  3. The knowledge worker must be able to remove an app from a tab.
  4. The knowledge worker must be able to undo the removal of an app from a tab.
  5. The knowledge worker must be able to drag and drop an app within a tab.
  6. The knowledge worker must be able to drag and drop an app onto another tab.
  7. The knowledge worker must be able edit the settings of an app.
  8. The knowledge worker must not be able to see an option to edit settings if none exist for an app.
  9. The knowledge worker must be able to minimize and restore an app.
  10. The knowledge worker must be able to maximize and restore an app.
  11. The system must remember the minimized and maximized states for an app.
  12. The knowledge worker must be able to refresh the content of an app.

User experience

Themes

Apply a theme to the start page

  1. Applies the selected theme and takes the knowledge worker to their start page. Feedback displays at top of page with a close option: ‘Theme has been applied’. Refer to the Action Feedback UI design pattern for additional details.
  2. Sorts the available themes by a recently added or popular views.
  3. The default sort view is recent.

Tabs

Create a tab

  1. Clicking on the + will add another tab. The + will not be present if there are 6 tabs created
  2. The default name of the tab when added will be New Tab; the knowledge worker must be able to type the desired tab name. Tabs only display the first 16 characters. Pressing enter will commit the name of the tab.

Default three column layout when creating a new tab

  1. A three column layout is the default layout when a new tab is added

Apply a layout to a tab

  1. Selecting “Change Layout” from the tab menu displays the change layout window.
  2. Carousel paging
  3. Hover state, clicking on the layout applies the layout to the tab and changes the currently selected layout image on the right.
  4. If the layout selected will be reducing the number of columns currently being used, the knowledge worker will be presented with a message confirming the movement of thier apps. Clicking ok will change the layout. Clicking cancel will return the user to the change layout page.

Re-arrage the order of the tabs

  1. The knowledge worker can reorder the tabs by dragging and dropping the tab into place. Dragging the title bar of an app displays a proxy image containing the name of the app. When the knowledge worker drags the app over an empty column a dashed box appears. When the proxy is released over the column the app appears in that column.

Rename a tab

  1. Clicking the tab menu icon displays a drop down menu.
  2. Link to rename the tab, when this is selected the name of the tab appears in an editable input box. Tab names are limited to 16 characters

Remove a tab

  1. Clicking the tab menu icon displays a drop down menu.
  2. Select the “Remove” option. Once the tab is removed, a delete conformation is displayed at the top of the page with an undo and close option: “[tab name] has been removed”. Refer to the Action Feedback UI design pattern for additional details.

Undo the removal of a tab

  1. Click undo
  2. The tab will be added back to the start page and all of the apps which were on the tab when it was removed will be returned as well

Apps

Adding an app

  1. The configure button should take the background color of the inactive tab and the text color of the inactive tab. There should also be a 1px line around the configure button that takes the color of the inactive tab text. Clicking the configure button displays the Configure page. Apps are added to the first column in the start page by default.
  2. Clicking the Add an App link displays the Configure page. Apps are added to the column on the start page where the Add an App link was selected.

  1. Adds the app to the first tab of the knowledge worker’s start page. Feedback displays at top of page with a close option: ‘App has been added’. Refer to the Action Feedback UI design pattern for additional details.
  2. The add button becomes inactive after the app has been added. Multiple apps can be added while on this page.
  3. Clicking will return the user to the start page.

Removal of an app

  1. Deletes app from the tab. Upon clicking the icon a pop up window is displayed asking the knowledge worker if they are sure they want to delete the app.

Undo the removal of an app

  1. After the app is deleted, feedback is displayed at the top of the page with an undo and close option: “App has been removed”. Refer to the Action Feedback UI design pattern for additional details.

Drag and drop an app within a tab or on antoher tab

  1. **Dragging the title bar of an app displays a proxy image containing the name of the app. Dragging the proxy over an inactive tab highlights the tab. Releasing the proxy over the tab activates the tab and places the app in the upper left hand corner of the tab.

Edit settings

  1. The edit settings icon is only present if the app has settings that are able to be edited

Resizing an app

  1. Minimizes the app
  2. Restores the app
  3. Opens app in canvas view (maximize)

1. App title 2. Restores the app.

Refreshing an app

  1. Refreshes app

Test plans

Themes

  1. The knowledge worker must be able to apply a theme to the start page.
    • Verify the default view when viewing themes in the gallery is all categories sorted by recent
    • Verify that when sorted by recent, the public dates on the themes reflect this
      • Verify the knowledge worker can apply a theme from this view
    • Verify the knowledge worker can view theme options by categories
      • Verify that when sorted by popular, the users number on the themes reflect this
      • Verify the knowledge worker can apply a theme from a specific category

Tabs

  1. The knowledge worker must be able to create up to six tabs on the start page.
    • Verify when there are 5 or less tabs, there is a tab with the +
    • Verify clicking the + tab results in a new tab being added
    • Verify a 16 character limit when naming a new tab
    • Verify when there are 6 tabs present; the + tab is not
  2. The knowledge worker must see a three-column layout as the default when a new tab is created.
    • Verify three columns are displayed by default
    • Verify the presence of a link to Add an App within each column
  3. The knowledge worker must be able to apply a layout to a tab.
    • Verify the change layout option on the tab drop down menu
    • Verify the ability to scroll through layout options
    • Verify when choosing a layout which has less columns than the tab currently has, a request to move apps automatically is presented.
    • Verify the ability to change the layout of a tab
    • Verify no apps are lost when layouts are changed
    • Verify the ability to close the change layout view
  4. The knowledge worker must be able to re-arrange the order of the tabs on the start page.
    • Verify the ability to drag and drop the order of the tabs on the start page
  5. The knowledge worker must be able to rename a tab.
    • Verify the ability to rename a tab
    • Verify a 16 character limit when renaming a tab
    • Verify there is an error message is the knowledge worker fails to enter a tab name
  6. The knowledge worker must be able to remove a tab unless it is the last one on the start page.
    • Verify the remove option is present on the tab drop down menu
    • Verify the ability to remove a tab
    • Verify a tab removal confirmation
    • Verify the remove option is not present on the tab drop down menu when it is the only tab (not counting the + tab)
  7. The knowledge worker must be able to undo the removal of a tab.
    • Verify the ability to undo the removal of a tab from the removal confirmation
      • Verify all of the apps which were present on the tab at the time of removal have also been restored

Apps

  1. The knowledge worker must be able see empty zones on a tab and an explicit action to add an app.
    • Verify empty zones on a tab are present when a new tab is added.
    • Verify there is a link to add an app
    • Verify there is a link to add an app when all apps have been removed from a column on an existing tab
  2. The knowledge worker must be able to add an app to a tab.
    • Verify the ability to press Configure and see all apps within the galley
    • Verify apps selected are added to the tab which was selected when the configure button was pressed
  3. The knowledge worker must be able to remove an app from a tab.
    • Verify an app can be removed from a tab by selecting the remove option
    • Verify there is confirmation that the app has been removed
  4. The knowledge worker must be able to undo the removal of an app from a tab.
    • Verify there is an option in the app removal confirmation to undo the action
    • Verify that the app is returned to the tab in the same location when undoing the removal
  5. The knowledge worker must be able to drag and drop an app within a tab.
    • Verify there is a visual indication as to which app is being moved
    • Verify there is a visual indication as to the placement of the app when it is dropped
    • Verify apps can be moved within a column
    • Verify apps can be moved between columns
  6. The knowledge worker must be able to drag and drop an app onto another tab.
    • Verify there is a visual indication as to which app is being moved
    • Verify there is a visual indication as to the placement of the app when it is dropped
  7. The knowledge worker must be able edit the settings of an app.
    • Verify the system displays an option to edit settings and that settings can be modified
  8. The knowledge worker must not be able to see an option to edit settings if none exist for an app.
    • Verify the system does not display an option to edit settings if none exist for an app
  9. The knowledge worker must be able to minimize and restore an app.
    • Verify an app can be successfully minimized
    • Verify an app can be successfully restored
  10. The knowledge worker must be able to maximize and restore an app.
    • Verify an app can be successfully maximized
    • Verify an app can be successfully restored
  11. The system must remember the minimized and maximized states for an app.
    • Verify that the minimized app is still minimized when returning to the Start Page
    • Verify that the maximized app is still maximized when returning to the Start Page
  12. The knowledge worker must be able to refresh the content of an app.
    • Verify the contents of the app can be refreshed
    • Verify only the app is refreshed async without reloading the entire page
