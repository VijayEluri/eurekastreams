Introduction

APIs Developer Integration Guidelines

Overview

This document will describe, for developers, how to use this API to integrate Eureka Connect into a website.

API Calls

API calls are made by inserting JSON request into a URL. The URL will need to follow the specific format below.

Example API Call

The following sections will walk a developer through creating an API call.

URL Format

The URL that calls the Eureka Connect API includes, is as follows:

host_name/api/api_version/read_only/api_name/JSON_request

host_name is the URL of the Eureka instance api is the literal string “api” api_version is the version number of the Eureka Connect API being used read_only can have the value of either full or read. Full means that all APIs are available, read means that only the READ-ONLY APIs are available.
Each of these settings can have different security configured. For example full can be OAUTH while read is Integrated Authorization. It is recommended that Integrate Authentication not be used for full, as this opens your system to potential security threats. api is the name of the API being called. Full descriptions of the APIs available are listed in the API section. JSON_request is the JSON request object

Creating a JSON Request Object

To make a call to the API the JSON object only requires one parameter, the request parameter. The JSON object, therefore, has the following appearance:

{“request”:7}

Where 7 is the request object as defined in the API description. If the API does not require a request parameter, the JSON object appears as follows:

{}

So, for example, suppose it was necessary to post the comment “That’s very helpful!” to an activity with an ID of 42. The API to use would be
[postActivtyCommentAction]. That API takes a CommentDTO as a request item. Specifically to comment what is needed is the activity ID and the comment
text. So the JSON object would have the following appearance:

{“request”:{“activityId”:42.”body”:”That’s very helpful!”}}

Encoding the JSON Request Object

Since the JSON Request Object will be part of the URL, it is important to encode the JSON Request Object so that no illegal characters are used in the
URL. In JavaScript the function encodeURIComponent() will execute the necessary transformations to the JSON request object so that no syntax errors will
occur in the URL.

Final API call URL

To complete our example lets use the post a comment request above along with the following assumptions:

URL to Eureka: http://demo.eurekastreams.org/ API Version: 0 Since posting to comment is making an edit to the stream, the read_only value to use is full. API name: postActivityCommentAction

So after encoding the JSON request object, the following is the complete API call URL:

http://demo.eurekastreams.org/api/0/full/postActivityCommentAction/%7B%22request%22%3A%7B%22activityId%22%3A42%2C%22body%22%3A%22That’s%20very%20helpful! %22%7D%7D

API Response

The response from the API is unencoded JSON of the APIs documented response type as described in the API section. If the response is a simple type
(e.g. Number, Date, String), the response will simply be that value in the appropriate type, otherwise the returned value will be a JSON object.

Please Note There may be additional data items in returned types than are explained in the documentation. Considered these reserved values that may
be removed or changed without notice. It is strongly recommended not to use these values.
